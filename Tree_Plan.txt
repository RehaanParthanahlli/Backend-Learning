# Bckend-Tree
```
HTML (the skeleton of a house)
│
├── CSS (the paint + interior design)
│    ├── Tailwind CSS (pre-made design blueprints)
│    └── Other frameworks (Bootstrap, Material UI = furniture catalogs)
|
├── JavaScript (the electricity + plumbing that makes things work)
│    ├── Vanilla JS (wiring it all by hand)
│    ├── React (modular appliances you can plug in/out)
│    │    └── Next.js (a smart home system: handles rooms, lighting, 
│    │                 even security + delivery = frontend + backend)
│    │
│    └── Other frameworks (Vue, Angular = different house-building styles)
│
├── Authentication (the locks + keys on your doors)
│    └── OAuth, JWT, NextAuth (different lock systems: keycards, PINs, biometrics)
│
├── API (the delivery service bringing supplies in/out of your house)
│    ├── REST (menu-based ordering: one endpoint per dish)
│    └── GraphQL (buffet: ask for exactly what you want in one go)
│
├── Node.js (the kitchen itself: lets you cook with JS outside the browser)
│    └── Express.js (the head chef + waitstaff: organizes orders, serves dishes)
│
├── ORM (the translator between chefs and storage room)
│    ├── Prisma / Sequelize / TypeORM (they speak “database” so you don’t have to)
│
└── Database (the pantry/storage room)
     ├── SQL (organized shelves with labels: MySQL, PostgreSQL)
     └── NoSQL (a giant walk-in freezer: MongoDB, Firebase)
```
